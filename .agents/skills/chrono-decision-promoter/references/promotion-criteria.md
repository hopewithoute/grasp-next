# Promotion Criteria

## Strong ADR signals

- authorization/security rule
- persistence ownership rule
- API contract standard
- service boundary
- distributed system behavior
- eventing/integration pattern
- deployment/runtime rule
- irreversible migration strategy
- cross-feature invariant

## Weak ADR signals

- one-off UI behavior
- feature-local copy
- temporary workaround
- unvalidated experiment
- minor naming convention
- isolated implementation detail

## Decision lifecycle

Feature-level decision:

- local, contextual, easier to change

ADR candidate:

- potentially system-wide, needs review

Accepted ADR:

- binding architecture memory

Superseded ADR:

- no longer current but retained as history
